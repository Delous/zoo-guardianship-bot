# Telegram-бот «Какое у вас тотемное животное?»

Telegram-бот для популяризации программы опеки Московского зоопарка. Бот проводит пользователя через викторину, считает баллы по ответам и показывает подходящее животное, которое можно поддержать через программу опеки.

Большие панды намеренно не используются в вопросах и результатах, потому что их нельзя взять под опеку.

## Возможности

- запуск через `/start`;
- викторина с несколькими вариантами ответов;
- хранение животных, вопросов, ответов, баллов и текстов в SQLite;
- автоматическое создание и заполнение базы при первом запуске;
- FSM и прогресс пользователя в SQLite;
- продолжение викторины после перезапуска бота;
- итоговое животное с описанием, фактом, блоком об опеке и изображением;
- Telegram share-ссылка для публикации результата;
- обращение к сотруднику с отправкой сообщения администратору;
- сбор оценки и необязательного текстового отзыва;
- fallback-сообщения для неподходящего ввода;
- логирование запуска, создания базы, старта и завершения викторины, обращений и отзывов.

## Установка

```bash
cd zoo_guardianship_bot
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

Для Linux/macOS активация окружения будет отличаться:

```bash
source venv/bin/activate
```

## Настройка `.env`

Скопируйте пример:

```bash
copy .env.example .env
```

Заполните значения:

```env
BOT_TOKEN=токен_от_BotFather
ADMIN_CHAT_ID=id_чата_администратора
BOT_USERNAME=username_бота_без_@
DATABASE_PATH=zoo_guardianship_bot.db
```

`BOT_USERNAME` используется для share-ссылки вида `https://t.me/share/url?...`.

## Запуск

```bash
python -m app.main
```

При старте приложение:

1. читает `.env`;
2. создаёт SQLite-файл по `DATABASE_PATH`, если его ещё нет;
3. создаёт таблицы;
4. заполняет таблицы начальными данными из `app/database/seed.py`;
5. запускает polling Telegram-бота.

## Как создаётся и заполняется SQLite

Схема и seed-данные находятся в `app/database/seed.py`. Функция `init_database()` из `app/database/connection.py` вызывает создание таблиц и заполнение базы.

Seed выполняется только если таблица `animals` пустая. Это защищает уже изменённую базу от повторной перезаписи при каждом запуске.

Тексты в seed-файле русскоязычные. Файл сохранён в UTF-8; при ручном редактировании сохраняйте ту же кодировку.

## Где менять данные викторины

Основные списки в `app/database/seed.py`:

- `ANIMALS` — животные, названия, описания, факты, тексты результата и имена изображений;
- `QUESTIONS` — вопросы, варианты ответов и начисляемые баллы;
- `STATIC_TEXTS` — приветствие, тексты опеки, contact/feedback/fallback и шаблон share-сообщения.

Если база уже создана, изменения seed-файла не применятся автоматически. Для учебного проекта проще удалить SQLite-файл и запустить бота заново.

## Алгоритм подсчёта

Каждый вариант ответа добавляет баллы только одному животному. Это задано в `QUESTIONS[*].answers[*].scores`.

После последнего вопроса бот:

1. берёт накопленные баллы пользователя из `user_quiz_progress.scores_json`;
2. выбирает животное с максимальной суммой;
3. при равенстве выбирает первое животное по порядку в таблице `animals`;
4. сохраняет результат в `user_quiz_progress.result_animal_id`.

## FSM в SQLite

В проекте используется aiogram 3.x и собственный storage `app/bot/sqlite_storage.py`.

FSM хранится в таблице `user_quiz_progress`:

- `telegram_user_id`;
- `current_state`;
- `fsm_data_json`.

В этой же таблице хранится прогресс викторины:

- `current_question_position`;
- `scores_json`;
- `selected_answers_json`;
- `started_at`;
- `finished_at`;
- `result_animal_id`.

## Продолжение после перезапуска

Так как state и прогресс сохранены в SQLite, перезапуск процесса не сбрасывает прохождение. Если пользователь уже начал викторину и снова отправляет `/start`, бот предлагает:

- продолжить с текущего вопроса;
- начать заново.

Если викторина завершена, бот предлагает посмотреть результат, оставить отзыв или пройти викторину снова.

## Изображения животных

Имена файлов заданы в `ANIMALS`:

- `manul.jpg`;
- `capybara.jpg`;
- `raccoon.jpg`;
- `flamingo.jpg`;
- `eagle_owl.jpg`.

Положите реалистичные изображения в папку:

```text
media/animals/
```

Файл должен называться так же, как `image_filename` в seed-данных. Если изображения нет, бот не падает: он залогирует предупреждение и отправит текст результата.

## Telegram share-ссылка

Шаблон текста хранится в базе как `share_text_template`. Поддерживаются плейсхолдеры:

- `{animal_name}`;
- `{bot_link}`.

Сервис `SharingService` формирует ссылку:

```text
https://t.me/share/url?url=...&text=...
```

`bot_link` строится из `BOT_USERNAME` в `.env`.

## Контактный механизм

После кнопки «Связаться с сотрудником» бот просит написать вопрос. Затем:

1. сохраняет обращение в таблицу `contact_requests`;
2. отправляет администратору сообщение в `ADMIN_CHAT_ID`;
3. передаёт имя пользователя, username, Telegram ID, итоговое животное, текст вопроса и дату обращения;
4. показывает пользователю подтверждение.

Тексты контактного сценария хранятся в `STATIC_TEXTS`: `contact_intro` и `contact_success`.

## Сбор отзывов

После кнопки «Оставить отзыв» бот:

1. предлагает оценку от 1 до 5 inline-кнопками;
2. просит необязательный комментарий;
3. позволяет нажать «Пропустить»;
4. сохраняет оценку и комментарий в таблицу `feedback`.
